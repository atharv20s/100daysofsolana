3. Comparison Reference Grid

Concept Traditional Database Solana Accounts
Data location Rows in tables on a centralized server Accounts on a distributed ledger across validators
Schema Defined by database engines (SQL schemas, tables) Defined by custom serialized bytes inside account data
Access control Application-level routes / SQL database roles Enforced by runtime: only owner program has edit access
Cost of storage Cloud monthly hosting costs, AWS billing Upfront rent-exempt deposit (query via RPC, fully refundable)
Identity/keys Auto-increment integers, UUIDs 32-byte public keys or Program Derived Addresses (PDAs)
Reads SQL queries, document lookups RPC calls (getAccountInfo, getProgramAccounts)
Writes INSERT/UPDATE statements in code Transactions containing signed authorization keys
Code vs data Program binary & database are separate systems Both are accounts, separated by the executable status
Deletion DELETE queries remove the record row Close instructions return the rent SOL deposit back to owner
Visibility Private by default; exposure requires APIs Public by default; readable globally without auth headers
